为什么我们需要 证据链？

100% 本地隐私

你的代码、Diff 和浏览记录永远不会离开你的电脑。没有云端上传，不需要账号注册。数据安全是我们的底线。

全自动行为采集

不仅是"你打开了什么窗口"，Project Mirror 还会监控文件系统的 Diff 变化。无论你是否 Commit，每一行代码变更都会被记录。

智能技能归因

基于你的实际行为生成技能树。AI 自动分析你的工作内容，告诉你这周是在"重构代码"还是在"摸鱼"。支持离线 LLM。

为 Windows 工程师打造

  • 系统托盘常驻: 极低资源占用，静默运行于后台。
  • Git Diff Watcher: 自动监听指定目录的文件变更，无需手动 Commit。
  • Local API: 内置本地 HTTP Server，支持自定义脚本扩展。
  • 离线可用: 无网环境下自动降级为规则模式，不影响数据记录。

快速开始

Project Mirror 是一个零配置的 Windows 托盘应用。只需 3 步即可开始构建你的工作证据链。

1. 安装

推荐使用 winget 安装最新版本，或直接从 GitHub Releases 下载 msi 安装包。

winget install project-mirror

2. 初始化

首次运行后，Project Mirror 会最小化到系统托盘。右键点击图标选择 "打开面板"。

  • 默认数据存储路径：%APPDATA%\ProjectMirror\db
  • 默认监听端口：127.0.0.1:4120

3. 配置监听目录

为了让 Diff Collector 生效，你需要在设置页添加你的工作区目录：

C:\Users\Dev\Projects\Go-Backend

隐私策略

最后更新：2023年10月24日

我们深知工程师对数据的敏感性。Project Mirror 遵循 "Local-First" 原则设计。

数据存储

所有采集的数据（包括但不限于应用活动、文件差异、浏览历史）均存储在您计算机的本地 SQLite 数据库中。我们没有服务器，也不会上传任何数据。

AI 处理

默认情况下，Project Mirror 使用离线规则引擎。如果您配置了 OpenAI API Key，数据仅在生成周报时发送给 LLM 提供商，且会经过 PII（个人敏感信息）脱敏处理。

配置指南

高级用户可以通过修改 config.json 来自定义采集行为。

忽略名单

在 ignore_apps 列表中添加不需要记录的应用进程名：

{
  "collectors": {
    "window": {
      "ignore_apps": ["Spotify.exe", "Steam.exe"]
    }
  }
}
